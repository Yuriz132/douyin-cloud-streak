# 抖音云端自动续火花 · 零基础完整教程（国内服务器版）

> **这是什么**：用一台云服务器，每天在你设定的时间（默认 **08:00**）自动给你的抖音好友发一条消息（默认 🔥），帮你把「火花 / 连续聊天天数」一直养下去——不用每天惦记手动发。
>
> **给小白的话**：本教程假设你**完全没接触过服务器、没写过代码**。每一步都写清楚了「点哪里、复制哪段命令、会看到什么」，跟着做就行。全程不需要人工智能帮你，自己照着做就能完成。

---

## 0. 能做什么 / 不能做什么

| ✅ 能做的 | ⚠️ 注意 / 限制 |
|----------|----------------|
| 每天定时自动给**指定好友**发「续火花」消息（内容可自定义） | 依赖抖音网页版的登录状态，登录态会过期（几天到几周），过期后要重新提取一次 |
| 指定任意好友昵称列表；找不到名字时自动用搜索兜底 | 抖音对「机房 IP + 异地登录」有风控，建议先用小号试几天 |
| 完全云端运行，关掉电脑也照常执行 | 需要一台 **24 小时开机**的云服务器（或长期开机的旧电脑） |
| 与服务器上其他项目**完全隔离**（独立目录 / 独立环境 / 独立定时任务） | 本仓库**不含**任何账号、密码、Cookie 等隐私 |

## 2. 第一步：买一台国内服务器（最好你所在城市）

### 2.1 为什么买「国内、最好同城」的服务器？（重点）

抖音判断「这到底是不是你本人在操作」，主要看**登录 IP 的特征**。理解下面三点，你就知道为什么别买境外机房节点、要买国内甚至同城：

1. **IP 是不是「机房 IP」**：云服务器的 IP 属于「数据中心 / 机房」段，不是家庭宽带。抖音对机房 IP 本来就比家庭宽带更敏感。
2. **IP 在哪个国家 / 城市**：如果你人常在**国内某市**，却从**境外机房**登录，等于同时踩中「境外 + 异地」两个风险点，最容易被判定为异常登录——轻则弹验证码，重则临时风控、发不出去。
3. **IP 是否和常用地一致**：选**你所在城市**的节点，登录 IP 的城市和你日常一致，行为看起来就像你本人在国内用，被风控的概率大幅下降；而且同城网络延迟低，抖音页面加载快，脚本更稳、不容易卡超时。

> ✅ **结论**：优先选**你所在城市**的国内节点；其次选国内任意节点；**尽量避免境外节点**。
> ⚠️ 但云服务器 IP 终究是「机房 IP」，不是你家宽带，所以仍建议**先用一个小号试跑几天**，确认不被风控、消息能正常发出，再换主号。想要最自然的 IP，见文末「备选方案：用家里电脑」。

### 2.2 去哪里买、怎么买（阿里云 / 腾讯云 / 华为云通用步骤）

国内主流厂商都行，下面以通用流程说明（任选一家）：

1. 打开对应官网，注册并实名（国内服务器购买需要实名，按页面提示上传身份证即可）。
2. 进入「云服务器 / ECS / CVM」控制台 → 创建实例。
3. 按 2.3 的选项填好。
4. 下单付款，等 1~2 分钟实例就创建好了。

### 2.3 购买时的关键选项

| 选项 | 怎么选 | 说明 |
|------|--------|------|
| **地域** | 选**你所在的城市**（如「华东 1（杭州）」「华北 2（北京）」等就近节点） | 见 2.1，同城最稳 |
| **实例规格** | 入门即可，1 vCPU / 2 GiB（如阿里云 `ecs.t6`、腾讯云「标准型 SA2 1核2G」） | 项目很轻量 |
| **镜像 / 系统** | **Ubuntu 24.04 64 位** | 本教程基于此系统 |
| **系统盘** | 20~40 GB 就行 | 几乎不占空间 |
| **公网带宽** | 按量付费 1~3 Mbps | 脚本很小，流量极少 |
| **登录方式** | 设置 **root 用户密码**（务必记牢） | 后面 SSH 登录要用 |
| **安全组** | 放行 **22 端口（TCP）** | 见 2.4 |

> 下单完成后，在实例列表里记下两样东西：**公网 IP 地址**（一串数字，如 `123.45.67.89`）和 **root 密码**。后面用得到。

### 2.4 放行 22 端口（让本机能连上服务器）

- 在实例的「安全组」设置里，确认**入方向**有一条允许 `22 / TCP`、来源 `0.0.0.0/0` 的规则（厂商一般默认带，没有就手动加一条）。
- 这一步只需要 SSH 登录用，不需要开放其他端口。

### 2.5 本机连上服务器（小白：什么是终端 / SSH）

- **Windows 用户**：装 [Git for Windows](https://git-scm.com/)，装完右键文件夹能选「Git Bash Here」，打开的就是命令行窗口。
- **macOS 用户**：打开「启动台 → 其他 → 终端」即可。

连接命令（把 `你的服务器IP` 换成你记下的公网 IP）：

```bash
ssh root@你的服务器IP
```

- 第一次会问 `Are you sure...` → 输入 `yes` 回车。
- 然后问密码 → 输入你设置的 **root 密码**（输入时屏幕上**不显示字符**，正常，输完回车即可）。
- 看到类似 `root@xxx:~#` 的提示符，说明已经连进服务器了。

## 3. 第二步：在本机提取抖音登录态（state.json）

### 3.1 为什么不在服务器上登录，而在本机？

抖音对「机房 IP + 异地登录」敏感，在服务器上直接登录容易弹验证码甚至封号；而**你本机**（家庭 / 手机网络）环境更自然，能正常弹出**二维码**让你扫码。所以原则是：

> **登录态在本机拿，服务器只复用，绝不在服务器上登录。**

### 3.2 本机安装 Python

- Windows / macOS 都去 [python.org](https://www.python.org/downloads/) 下载 **Python 3.10 以上** 安装包。
- 安装时 **务必勾选「Add Python to PATH」**（Windows），然后一路下一步。
- 验证：打开命令行，输入下面任一行，能显示版本号就成功：

```bash
python --version
# 或
python3 --version
```

### 3.3 本机安装 Git（Windows）

- 去 [git-scm.com](https://git-scm.com/) 下载安装，一路默认下一步即可。
- macOS 一般自带 Git；没有就装 [Xcode 命令行工具](https://developer.apple.com/xcode/)：`xcode-select --install`。

### 3.4 把本仓库的文件拿到本机

两种办法，选一种：

- **办法 A（最简单，不用 Git）**：在 GitHub 页面点绿色 **「Code」→「Download ZIP」**，解压到任意文件夹（如 `D:\douyin-cloud-streak`）。
- **办法 B（用 Git）**：
  ```bash
  git clone https://github.com/你的用户名/douyin-cloud-streak.git
  ```

> 之后所有「在本机执行」的命令，都先 `cd` 进这个文件夹，例如：`cd D:\douyin-cloud-streak`（Windows）或 `cd ~/douyin-cloud-streak`（macOS）。

### 3.5 安装依赖

在本机仓库文件夹里执行：

```bash
pip install -r requirements.txt
# 如果上面报错，试： python -m pip install -r requirements.txt
```

### 3.6 运行提取工具，扫码登录

**Windows**：直接双击仓库里的 **`1.本地提取通行证.bat`**。
**macOS / 手动**：

```bash
python extract_cookie.py
```

会弹出一个浏览器窗口，请：
1. 用**扫码 / 短信验证码**登录你的抖音网页版账号；
2. 登录成功后，回到命令行窗口按 **回车键**；
3. 脚本把登录态保存为当前目录下的 **`state.json`**。

> ⚠️ `state.json` 含有登录凭证，等同于账号密码。**千万不要把它提交到任何公开仓库**（本仓库已用 `.gitignore` 屏蔽）。

## 4. 第三步：把文件传到服务器

### 4.1 打包项目代码（不含 state.json）

在本机仓库文件夹执行（Windows 用 Git Bash）：

```bash
zip -r ../douyin-cloud-streak-deploy.zip cloud_streak.py config.json requirements.txt deploy_on_server_safe.sh
```

> 小白解释：`zip` 是把几个文件压成一个包，方便一次传过去。「`../`」表示压到上层目录，生成 `douyin-cloud-streak-deploy.zip`。

### 4.2 上传到服务器

把 `你的服务器IP` 换成你的真实 IP：

```bash
# 1) 代码包传到服务器的 /root/
scp ../douyin-cloud-streak-deploy.zip root@你的服务器IP:/root/

# 2) 登录态 state.json 先传到 /root/（部署完再挪进项目目录）
scp ./state.json root@你的服务器IP:/root/
```

> 命令说明：`scp` 是「安全复制」，把本机文件传到服务器。`root@IP:路径` 表示「传到这台服务器的某个路径」。第一次传会让你确认一次，输入 `yes` 回车即可。

## 5. 第四步：服务器一键部署

> **隔离原则（重点）**：部署脚本**绝对不影响你服务器上已有的任何项目**（比如你正在跑的网站、其他服务）。它只用**独立目录 `/opt/douyin-cloud-streak`**、**独立 Python 环境**、**独立定时任务**，不修改系统 Python，不卸载 / 升级任何现有软件。

### 5.1 登录服务器

```bash
ssh root@你的服务器IP
```

### 5.2 执行部署脚本

```bash
sudo bash /root/deploy_on_server_safe.sh
```

脚本会自动完成（耗时几分钟，期间会下载浏览器，请耐心等）：
1. 安全检查（和现有目录不冲突才继续）；
2. 内存不足时自动建 **2G 交换空间**，防止内存溢出；
3. 仅安装必要系统库（已适配 Ubuntu 24.04）；
4. 解压代码到 `/opt/douyin-cloud-streak`；
5. 建**独立 Python 环境**并安装依赖、下载浏览器内核；
6. 设置**中国时区**（保证时间是你本地时间）；
7. 写入**独立定时任务**：每天 **08:00** 自动运行。

### 5.3 把登录态放进项目目录

```bash
mv /root/state.json /opt/douyin-cloud-streak/state.json
```

## 7. 第六步：日常维护（Cookie 过期了怎么办）

登录态（`state.json`）通常几天到几周会过期，过期后脚本会失败并截图 `error_screenshot.png`。恢复只需 2 步：

1. **本机**重新运行 `1.本地提取通行证.bat`（或 `python extract_cookie.py`）生成最新 `state.json`；
2. **本机**运行同步脚本把新登录态传上去（先确认脚本顶部 `SERVER_IP` 填的是你的服务器 IP）：
   ```bash
   bash sync_state_safe.sh
   ```
下次定时任务会自动用新的登录态，无需手动重启。

## 8. ★ 自定义：发给谁 / 发什么 / 几点发（核心）

这三样全部由**服务器上的 `config.json`** 控制，改完保存即可，**下次运行自动生效，不用重启服务**。

### 8.1 自定义「发给谁」

打开 `/opt/douyin-cloud-streak/config.json`（用 `nano config.json` 或把文件下载到本机改完再传回）：

```json
{
  "message": "🔥",
  "target_names": ["张三", "李四", "王五"],
  "max_count": 10
}
```

- `target_names` 里写你想发的**抖音昵称**，几个人都行，用逗号隔开，每个名字加英文引号。
- **昵称要和抖音里一模一样**（大小写、空格、特殊符号都要一致），否则可能找不到；找不到时脚本会自动用搜索兜底。
- 如果留空 `[]`，则按私信列表顺序发给前 `max_count` 个人。

> 怎么拿到准确昵称：打开抖音 App → 消息 → 找到那个好友，头像下面的名字就是昵称，照抄即可。

### 8.2 自定义「发什么内容」

还是改 `config.json` 里的 `message`：

```json
"message": "🔥"            // 只发火
"message": "早安~"         // 发文字
"message": "🔥在吗"        // 火 + 文字混用
```

- 支持 emoji，也支持普通文字。
- 保存后下次运行就用新内容。

### 8.3 自定义「几点发」

发送时间由服务器的**定时任务（cron）**控制。当前默认是 **每天 08:00**。

**cron 时间格式**（5 个字段）：

```
分 时 日 月 星期
*  *  *  *  *
```

- `*` 表示「每」；数字表示具体值。
- 前两个数最重要：**第 1 个是「分钟」，第 2 个是「小时」（24 小时制）**。

**常用时间速查表**：

| 想什么时候发 | 写成（前两位是 分 时） |
|--------------|------------------------|
| 每天 08:00（默认） | `0 8 * * *` |
| 每天 08:30 | `30 8 * * *` |
| 每天 12:00 | `0 12 * * *` |
| 每天 21:00 | `0 21 * * *` |
| 每天早上 07:15 | `15 7 * * *` |
| 只在工作日 23:30 | `30 23 * * 1-5` |

**怎么改时间**（在服务器上）：

```bash
ssh root@你的服务器IP
crontab -e
```

- 找到这一行（前面是 `0 8`）：
  ```
  0 8 * * * cd /opt/douyin-cloud-streak && /opt/douyin-cloud-streak/venv/bin/python /opt/douyin-cloud-streak/cloud_streak.py >> /opt/douyin-cloud-streak/run.log 2>&1
  ```
- 只改**最前面两个数**（分钟、小时）为你想要的时间，例如改成每天 21:00 就写 `0 21`。
- 保存退出：若用 `nano` 编辑器，按 `Ctrl+O` 再回车，然后 `Ctrl+X`；若用 `vim` 按 `Esc` 输入 `:wq` 回车。
- **改完立即生效，不用重启**。下次就会在新时间运行。

> **💡 自动重试（防设定时间那一瞬间网络抖断）**：脚本内置了失败自愈——如果某次运行**一条消息都没发出去**（典型原因是那一瞬间服务器到抖音的网络抖动），会自动隔 **2 分钟**再试，最多 **3 次**；只要发出过任意一条就停止重试，**不会重复发消息**。所以即使偶尔网络抽风，一般也能在几分钟内自动补发成功，不必半夜起来手动跑。
> （重试参数在 `cloud_streak.py` 顶部：`MAX_ATTEMPTS = 3`、`RETRY_GAP = 120`，想调更保守/更激进直接改这两个数即可。）

## 10. 常见问题 / 排错

| 现象 | 原因 / 解决 |
|------|------------|
| `chromium: error while loading shared libraries` | 系统库不全，重跑 `source venv/bin/activate && playwright install-deps chromium` |
| `❌ 未找到 state.json` | 登录态没放进 `/opt/douyin-cloud-streak/`，确认 `ls /opt/douyin-cloud-streak/state.json` |
| 任务在半夜（非设定时间）跑 | 服务器时区不对，`timedatectl set-timezone Asia/Shanghai` 后执行 `service cron restart` |
| 抖音弹验证码 / Cookie 过期 | 看 `error_screenshot.png`，本机重提取 `state.json` 后 `sync_state_safe.sh` |
| 发送 0 条 / 好友找不到 | 检查 `config.json` 的 `target_names` 是否与抖音昵称**完全一致** |
| 机房 IP 被风控 / 收不到消息 | 换同城 / 同省节点，或改用「家里长期开机的电脑」运行（见下） |

## 11. 安全与风控提示（务必先看）

- **机房 IP 风控**：云服务器 IP 属数据中心段，抖音对「机房 + 异地」敏感。**强烈建议先用小号试跑几天**，确认不被风控、消息能正常发出，再换主号。
- **登录态安全**：`state.json` = 你的抖音登录凭证，等同于账号密码。**绝不要**提交到公开仓库（本仓库已用 `.gitignore` 屏蔽）。
- **频率控制**：脚本已加入随机等待与隐身参数，但仍属自动化行为，请控制好友数量与发送频率，避免触发风控。
- **合规使用**：仅用于维持你本人与好友的正常互动，勿用于骚扰、营销等违规用途。

### 备选方案：用家里长期开机的电脑（IP 最自然）

如果你有一台**长期不关机**的旧电脑 / 笔记本 / 树莓派，直接在本机装好 Python 和依赖、运行 `cloud_streak.py`，用 `任务计划程序`（Windows）或 `crontab`（macOS / Linux）定时执行即可。家庭宽带 IP 比机房 IP 自然得多，风控风险最低。本教程的脚本在本地同样适用，只是少了「传到服务器」那几步。
